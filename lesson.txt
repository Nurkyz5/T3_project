# Очистка данных — это процесс приведения «сырого» датасета к чистому, структурированному и пригодному для анализа виду.

 📌 Цель:

 убрать ошибки
 сделать данные согласованными

Самые частые проблемы при работе с данными:
🔹 Какие бывают «грязные» данные

1. Пропущенные значения (Missing values)
    -есть датафрейм с сотрудниками, у кого есть столбцы - name, age, city, salary, и есть 10 сотрудников.  У 5 записи(Иван Иванов) отсутствует информация о его городе. 
Примеры:

пустые ячейки
NaN
None

👉 Что делать:

df.isnull().sum()
df.dropna()
df.fillna(0)
df.fillna(df.mean())

2. Дубликаты (Duplicates)
    -повторение той или иной записи в датафрейме (Иван иванов и информация в нём продублировалась в 6 записи)
👉 Удаление:

df.duplicated()
df.drop_duplicates()

3. Выбросы (Outliers)
    -неправдивая информация в определённом столбце у записи

👉 Пример: зарплата = 1 000 000 при среднем 30 000

👉 Методы:

IQR
Z-score
import numpy as np

z = np.abs((df['col'] - df['col'].mean()) / df['col'].std())
df = df[z < 3]

4. Неправильные типы данных
    -разный формат данных, например:
    -1) "Катя",17, Bishkek",10000
    -2) "Тима",22, Бишкек, 20000
    -3) "Иван", 30,biskek, 100000
    
Пример:

возраст = "двадцать"
дата как строка

👉 Исправление:

df['age'] = df['age'].astype(int)
df['date'] = pd.to_datetime(df['date'])

5. Несогласованные значения

Пример:

"USA", "usa", "Usa"
df['country'] = df['country'].str.lower()

============Работа с пропусками==========

Есть 3 основных вида работы с пропусками:
1) удалить
    - если данные не критичны для удаления
        -если мало пропусков
    -если строка полностью сломана
        -все её записи в столбцах содержат пропуски, аномалии и удаления этой записи почти необходимо и не критично
2) заполнить
    -если данные важны
    -если много пропусков
3) предсказать



Виды пропусков:
1) Случайные
2) Неслучайные


Можно удалять:
1) строки - если 50%+ пропусков
2) столбцы - если -5% пропусков
